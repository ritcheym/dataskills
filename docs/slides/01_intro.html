What is R?

R is a programming language designed for statistics and data analysis.

Think of R as the engine of a car — it does all the work under the hood.

Why R?

  • Free and open-source
  • Thousands of add-on packages
  • Massive community (especially in psychology!)
  • Makes your work reproducible

What is RStudio?

RStudio is an Integrated Development Environment (IDE) — it makes working with R much easier.

If R is the engine, RStudio is the dashboard, steering wheel, and GPS all in one.

Warning

Always launch RStudio, not R directly!

📦 Packages

What are Packages?

R comes with built-in functions (“base R”) — like the default apps on your phone.

Packages are add-ons created by the R community — like apps you download from the App Store.

There are thousands of packages for:

  • Data visualization 📊
  • Machine learning 🤖
  • Web scraping 🌐
  • Interactive dashboards 📱
  • Even playing Sudoku! 🧩

Install vs. Load: A Key Distinction

install.packages()

📲 Like downloading an app

  • Do it once
  • Only from the console
  • Stays until you remove it

library()

🚀 Like launching an app

  • Do it every session
  • Put it in your script
  • Only lasts until you close R

Activity: Install Your First Packages

Run these in the console (not a script!)

install.packages("tidyverse")
install.packages("beepr")

Important!

Never put install.packages() inside a script — only run it from the console. Why? You might accidentally reinstall and update a package, which could break your code.

Functions

Anatomy of a Function

A function is a named piece of reusable code. It takes arguments (inputs) and produces output.

function_name(argument1 = value1, argument2 = value2)

For example:

beep(sound = 8)
#     ^         ^
#     |         |
#  argument   value

Getting Help

There are several ways to get help with a function:

# If the package is loaded
?beep

# If you're not sure about the package
??beep

# The full specification
?beepr::beep

Every help page has the same structure: Description → Usage → Arguments → Details → Examples

Tab Auto-Complete

One of RStudio’s best features! Start typing a function name and press Tab:

  • RStudio will show you matching functions
  • Select a function and press Tab again to see its arguments
  • Each argument comes with a brief description

Try it!

Type sam in the console and press Tab. Select sample( and press Tab again to see the arguments.

Arguments In Practice

Example: sample()

# Randomly pick 5 letters
sample(x = letters, size = 5)
[1] "h" "z" "n" "w" "c"
# Allow duplicates
sample(x = letters, size = 5, replace = TRUE)
[1] "t" "y" "o" "n" "k"

Named vs. Positional Arguments

These two lines do the same thing:

# Named arguments (explicit)
sample(x = letters, size = 5, replace = TRUE)

# Positional arguments (implicit)
sample(letters, 5, TRUE)

With named arguments, order doesn’t matter:

sample(size = 5, replace = TRUE, x = letters)

Note

When learning, use named arguments — they make your code easier to read!

Objects

Creating Objects

Objects store things — numbers, text, dates, results. Use <- to assign values:

name <- "Maureen"
age <- 42
today <- Sys.Date()
halloween <- as.Date("2026-10-31")

To see what’s inside an object, type its name:

name
[1] "Maureen"
today
[1] "2026-04-06"

Types of Data

Object Type Key Rule
name <- "Maureen" Character (text) Must use quotes " "
age <- 42 Numeric (number) No quotes
today <- Sys.Date() Date (dynamic) Changes each day!
halloween <- as.Date("2026-10-31") Date (fixed) Wrapped in as.Date()

Check the Environment pane — your objects should appear there!

Objects Can Interact

name <- "Maureen"
age <- 42
today <- Sys.Date()
halloween <- as.Date("2026-10-31")

decade <- age + 10
full_name <- paste(name, "Ritchey")
how_long <- halloween - today
decade
[1] 52
full_name
[1] "Maureen Ritchey"
how_long
Time difference of 208 days

Google + “R” or “tidyverse” Stuck on a concept
AI tools (responsibly!) Need examples or explanations
Package vignettes Want a tutorial-style guide
Your classmates! Always a good idea

A Note on AI

AI tools can help with coding — but be careful while you’re building foundational skills.

Using AI to do your exercises is like getting a forklift to lift weights at the gym for you. It defeats the purpose!

Build your skills first, then use AI to accelerate your work — not replace your learning.

Wrapping Up

Key Takeaways

  • R is the language; RStudio is the tool that makes it easier
  • Always start with a clean workspace
  • install.packages() = download once; library() = load each session
  • Functions take arguments; use ? to learn about them
  • Objects store your data; use <- to create them
  • When stuck: restart R, check help, Google it, ask a friend!
